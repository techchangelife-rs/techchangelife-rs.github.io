Vue 3 源码解析系列（开篇）：总体架构与模块地图
2026-01-14|技术Vue3源码解析系列

为什么要读源码

  • 构建“可预测”的心智模型：遇到异常能快速定位
  • 掌握边界与约束：写出更稳健的组件与库
  • 形成“可迁移”的能力：不同框架的共性与差异

总体架构鸟瞰

Vue 3 的源码可分为三大部分：

  • reactivity：基于 Proxy 的响应式系统，包含 effect、ref、reactive、computed
  • runtime：运行时（核心与 DOM）渲染与更新逻辑，包含 vnode、renderer、scheduler
  • compiler：编译器，将模板转成渲染函数，包含 parser、transform、codegen

三者协同路径：
模板 → 编译器生成 render → 运行时根据响应式依赖驱动渲染 → DOM 更新

架构概览图：

模块地图

  • reactivity
    • 响应式依赖跟踪：track / trigger
    • 细粒度更新：Effect 调度与优先级
  • runtime-core
    • 虚拟节点：VNode 结构与 patch 策略
    • 组件实例：setup、生命周期与上下文
    • 调度器：queueJob、flushJobs
  • runtime-dom
    • 平台适配：DOM 操作与事件系统
  • compiler-core
    • AST：parse 生成抽象语法树
    • 转换：transform 注入指令与优化标记
    • 代码生成：codegen 输出渲染函数

学习建议

  • 自上而下：先建立模块地图，再进入子模块细节
  • 以问题驱动：从“为什么 update 这么快？”切入到 patch 与 scheduler
  • 读+写结合：用最小示例验证理解，如手写一个 track/trigger

系列规划（草案）

  1. reactivity：依赖收集与触发更新的细节
  2. runtime-core：VNode 与 patch 策略
  3. scheduler：任务队列与微任务机制
  4. compiler-core：AST、transform 与优化标记
  5. runtime-dom：事件与属性更新
  6. 综合案例：从模板到 DOM 的全链路
